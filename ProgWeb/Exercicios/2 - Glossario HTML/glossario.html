Evolução do HTML

Início e Primeiras Versões (1991 - 1999)

HTML 1.0 (1991):
Criado por Tim Berners-Lee, era extremamente simples, com o objetivo de compartilhar documentos científicos. Incluía apenas tags básicas como h1 para títulos, p para parágrafos e a para links. A ideia era focar no conteúdo, não na apresentação.
HTML 2.0 (1995):
Padronizada pela IETF (Internet Engineering Task Force), essa versão introduziu a possibilidade de formulários (form, input) e a inclusão de imagens (img). O foco ainda era em documentos, mas com maior interatividade.
HTML 3.2 (1997):
Esta versão trouxe elementos para a formatação de tabelas (table) e a formatação de texto. No entanto, ela carecia de um padrão estrito e muitos navegadores implementavam funcionalidades de forma própria, o que gerava problemas de compatibilidade.

A Era da Padronização e XHTML (2000 - 2004)

HTML 4.01 (1999):
O W3C (World Wide Web Consortium) assumiu o papel de padronização, e esta versão foi um marco. Ela separou a estrutura (HTML) da apresentação (CSS), um conceito revolucionário. Introduziu tags como div e span e tornou-se a base para o desenvolvimento web por muitos anos.
XHTML (2000):
O W3C tentou unificar HTML e XML para criar um padrão mais rigoroso. XHTML (eXtensible HyperText Markup Language) exigia que o código fosse bem-formado (tags sempre fechadas, por exemplo) para garantir maior compatibilidade e consistência. Apesar de popular por um tempo, sua rigidez e a falta de retrocompatibilidade com códigos HTML mal-formados levaram a comunidade a buscar uma alternativa mais flexível.

A Ascensão do HTML5 (2004 - Presente)

O Grupo WHATWG:
Enquanto o W3C focava em XHTML, um grupo de desenvolvedores (a Apple, Mozilla e Opera) formou o WHATWG (Web Hypertext Application Technology Working Group) com o objetivo de desenvolver um padrão mais prático e retrocompatível. Eles queriam uma linguagem que atendesse às necessidades de aplicações web modernas, não apenas documentos.
HTML5 (2014):
O W3C e o WHATWG uniram forças, resultando na versão final do HTML5. Este foi um grande salto, trazendo inúmeras novidades:
Novas tags semânticas: Tags como header, footer>, nav, article e section foram introduzidas para dar mais significado ao conteúdo e melhorar a acessibilidade e o SEO. Recursos multimídia nativos: Eliminou a necessidade de plugins como o Flash, com tags como video, audio e canvas. APIs poderosas: Introduziu APIs para geolocalização, armazenamento local (Local Storage) e interação com arquivos.
